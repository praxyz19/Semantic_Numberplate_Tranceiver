OBJECTIVE / INTRODUCTION

The objective of this work is to transmit license-plate semantics instead of full images so that recognition and reconstruction can be performed under bandwidth-constrained and noisy conditions.

The system combines plate detection, format-aware OCR, semantic transmission, and knowledge-base-guided reconstruction.

  • Detect the plate region using heuristic and YOLO-based methods.
  • Extract the character sequence using strict format validation.
  • Transmit only semantic information over an AWGN channel.
  • Reconstruct the plate using prior knowledge and template rendering.

LITERATURE SURVEY

Conventional ALPR pipelines rely on image transmission and generic OCR. Recent work improves detection with neural models, while plate-specific OCR systems improve recognition on constrained formats. Semantic communication motivates transmitting only task-relevant information, and federated learning supports distributed model improvement without centralizing all data.

  • Object detection models improve plate localization under clutter and blur.
  • Specialized OCR systems outperform generic OCR on structured plate text.
  • Semantic communication reduces bandwidth by transmitting only useful features.
  • Prior-based correction helps resolve character ambiguity in noisy recognition.

RESEARCH GAP / PROBLEM STATEMENT / NOVELTY

The research gap is that most OCR pipelines treat license plates as unconstrained text, even though plates follow a fixed format. This causes garbage reads such as random alphanumeric strings to be accepted as valid outputs.

The novelty of this work is to enforce the plate format at every stage of the pipeline.

  • Strict format validation using the plate regular expression.
  • Multi-model OCR fusion with fallback selection.
  • Character segmentation and template matching for difficult crops.
  • Knowledge-base priors for state codes and role-based character repair.

METHODOLOGY (PROPOSED)

Input Image ↓ Plate Detection ↓ Format-Aware OCR Fusion ↓ Semantic Packet Construction ↓ AWGN Channel Transmission ↓ Receiver Repair and Reconstruction
  • Detection: heuristic filtering and YOLO-based localization.
  • OCR fusion: Fast-Plate-OCR, EasyOCR, segmentation, Tesseract.
  • Transmission: character symbols encoded and sent over a noisy channel.
  • Reconstruction: knowledge-base priors and template-based rendering.

⚠️ Challenges & Solutions

Challenge 1: Low OCR Accuracy

Root Cause: Poor image quality, character ambiguity (O/0, I/1, L/1)

Solution: Aggressive preprocessing (2.5× contrast, 2× sharpness), character-level template matching, KB prior coercion

Challenge 2: Invalid Plate Reads

Root Cause: OCR reads garbage sequences violating format

Solution: Strict format validation; invalid reads rejected (score 0.05); only valid reads advance

Challenge 3: Training Time

Root Cause: 40K+ images × 4 clients × 10 rounds on CPU

Solution: Asynchronous federated learning; parallelized training overnight

Challenge 4: Character Confusion

Root Cause: Similar glyphs (S/5, G/6/9, B/8)

Solution: Position-based character role enforcement (only letters in pos 0-1,4-5; only digits in 2-3,6-9)
